Agent Class¶

This class represents a thread, of which a bot can have up to three. These agents operate on a bot based on how many boundaries the bot is on, and run algorithms entirely separate from each other, even if two of them are on the same boundary. No data is shared between these agents while inside a bot.

Class Definition¶

class agent.Agent(initialized: numpy.uint8 = 0, predecessor: str = None, successor: str = None, current_node: object = None, bot: object = None, agent_id: numpy.uint8 = None, successor_node: object = None, predecessor_node: object = None, successor_bot: object = None, predecessor_bot: object = None, link: object = None, link_published: numpy.uint8 = 0, links_tested: numpy.uint8 = 0, links_established: numpy.uint8 = 0, successor_link: object = None, segment_setup_status: numpy.uint8 = 0, candidate: numpy.uint8 = 0, identifier_token: object = None, delimiter: object = None, delimiter_passed: object = None, identifier_setup_status: numpy.uint8 = 0, wait_time: numpy.uint8 = None, wait: numpy.uint8 = None)¶

This class is responsible for an agent’s attributes and functions. Since most vars are established in or for certain phases, they may be denoted with an abbreviation of where they are created or primarily used.

BS: Boundary Setup SS: Segment Setup IS: Identifier Setup IC: Identifier Comparison SV: Solitude Verification BI: Boundary Identification

Parameters

  • initialized (uint8 0|1) – Has this agent called self.initialize()? Defaults to 0

  • predecessor (agent.Agent) – Agent which precedes this one in the DAG. Defaults to None

  • successor (agent.Agent) – Agent which succeeds this one in the DAG. Defaults to None

  • current_node (node_skeleton.Node) – Node which this agent’s host bot is on.

  • bot (bot_skeleton.Bot) – Host bot of this agent.

  • agent_id (uint) – This agent’s ID number. Defaults to None.

  • successor_node (node_skeleton.Node) – Reference to the succeeding Agent’s node.

  • predecessor_node (node_skeleton.Node) – Reference to the preceding Agent’s node.

  • successor_bot (bot_skeleton.Bot) – Reference to the succeeding Agent’s host bot.

  • predecessor_bot (bot_skeleton.Bot) – Reference to the preceding Agent’s bot.

  • link (link.Link) – BS. Shared memory space between two neighbors. Defaults to None.

  • link_published (uint8 0|1) – BS. Once this agent’s Link has been initialized, this becomes 1. Defaults to 0.

  • links_tested (uint8 0|1) – BS. The agents has tested its connection to the Link. Defaults to 0.

  • links_established (uint8 0|1) – BS. The agent has established connection with its neighbor at the end of the link. Default to 0.

  • successor_link (link.Link) – BS. Reference to the link of the successor agent in the DAG. Defaults to None.

  • candidate_coin_flipped (uint8 1|0) – SS. Has this agent flipped a coin yet to determine candidacy? Defaults to 0.

  • candidate_coin_flip_result (uint8 1|0) – SS. Is this agent a candidate as a result of the coin flip? Defaults to 0.

  • identifier (uint8 1|0) – IS. The digit of the reverse identifier in the sequence. Separate from the original coin flip.

boundary_setup()¶

Phase of the Leader Election Algorithm.

Establishes Link objects with neighbors if they are a predecessor or successor. One Link object is shared between two instances of an agent.

initialize(predecessor=None, successor=None, current_node=None, bot=None, agent=None)¶

Initialize this agent. Sets self.initialized to 1. Note successor and predecessor can come from the same directional keyword port, but will almost always be two different agents.
